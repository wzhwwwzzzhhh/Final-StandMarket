# 订单管理接口文档

## 1. 接口概述

本文档描述了用户端订单管理相关的API接口，包括订单的创建、查询、支付、取消和确认收货等功能。

## 2. 接口列表

| 接口名称 | URL | 方法 | 功能描述 |
|---------|-----|------|----------|
| 创建订单 | /user/order/create | POST | 创建新订单 |
| 获取订单列表 | /user/order/list | GET | 获取用户订单列表 |
| 获取订单详情 | /user/order/detail/{id} | GET | 获取订单详细信息 |
| 取消订单 | /user/order/cancel/{id} | PUT | 取消订单 |
| 支付订单 | /user/order/pay/{id} | PUT | 支付订单 |
| 确认收货 | /user/order/confirm/{id} | PUT | 确认收货 |

## 3. 接口详情

### 3.1 创建订单

**URL**: `/user/order/create`

**方法**: `POST`

**功能**: 创建新订单，同时保存订单详情

**请求参数**:

| 参数名 | 类型 | 必选 | 描述 |
|--------|------|------|------|
| amount | number | 是 | 订单总金额 |
| remark | string | 否 | 订单备注 |
| items | array | 是 | 订单详情列表 |
| items[].name | string | 是 | 商品名称 |
| items[].image | string | 否 | 商品图片 |
| items[].productId | number | 是 | 商品ID |
| items[].number | number | 是 | 商品数量 |
| items[].amount | number | 是 | 商品金额 |

**请求示例**:

```json
{
  "amount": 199.99,
  "remark": "购物车结算",
  "items": [
    {
      "name": "时尚T恤",
      "image": "http://example.com/tshirt.jpg",
      "productId": 1,
      "number": 2,
      "amount": 99.99
    }
  ]
}
```

**响应参数**:

| 参数名 | 类型 | 描述 |
|--------|------|------|
| code | number | 响应状态码，1表示成功，0表示失败 |
| msg | string | 响应消息 |
| data | object | 订单信息 |
| data.id | number | 订单ID |
| data.number | string | 订单号 |
| data.status | number | 订单状态 |
| data.userId | number | 用户ID |
| data.orderTime | string | 下单时间 |
| data.amount | number | 订单金额 |

**响应示例**:

```json
{
  "code": 1,
  "msg": "订单创建成功",
  "data": {
    "id": 1,
    "number": "ORD1776322495453",
    "status": 1,
    "userId": 20,
    "orderTime": "2026-04-16T14:54:55.453600400",
    "amount": 199.99
  }
}
```

## 4. 订单状态说明

| 状态码 | 状态名称 | 描述 |
|--------|----------|------|
| 1 | 待付款 | 订单已创建，等待用户支付 |
| 2 | 待发货 | 订单已支付，等待商家发货 |
| 3 | 待收货 | 商家已发货，等待用户收货 |
| 4 | 已完成 | 用户已确认收货，订单完成 |
| 5 | 已取消 | 订单已取消 |

## 5. 错误码说明

| 错误码 | 错误消息 | 描述 |
|--------|----------|------|
| 0 | 订单不存在或无权操作 | 订单ID不存在或不属于当前用户 |
| 0 | 网络错误，请稍后重试 | 网络连接失败或服务器异常 |

## 6. 前端调用示例

```javascript
// 创建订单
orderApi.createOrder({
  amount: 199.99,
  remark: '购物车结算',
  items: [
    {
      name: '时尚T恤',
      image: 'http://example.com/tshirt.jpg',
      productId: 1,
      number: 2,
      amount: 99.99
    }
  ]
}).then(response => {
  if (response.data.code === 1) {
    console.log('订单创建成功:', response.data.data)
  } else {
    console.error('订单创建失败:', response.data.msg)
  }
})

// 获取订单列表
orderApi.getOrderList(1).then(response => {
  if (response.data.code === 1) {
    console.log('订单列表:', response.data.data)
  } else {
    console.error('获取订单列表失败:', response.data.msg)
  }
})

// 支付订单
orderApi.payOrder(1).then(response => {
  if (response.data.code === 1) {
    console.log('支付成功')
  } else {
    console.error('支付失败:', response.data.msg)
  }
})
```

## 7. 后端实现说明

### 7.1 核心类

- **UserOrderController**: 处理用户订单相关的HTTP请求
- **OrderServiceImpl**: 实现订单相关的业务逻辑
- **OrderMapper**: 订单数据访问接口
- **OrderDetailMapper**: 订单详情数据访问接口

### 7.2 数据模型

- **Orders**: 订单主表，存储订单基本信息
- **OrderDetail**: 订单详情表，存储订单中的商品信息

### 7.3 业务流程

1. **创建订单**: 保存订单主表 → 保存订单详情表
2. **获取订单列表**: 查询订单主表 → 为每个订单加载订单详情
3. **获取订单详情**: 查询订单主表 → 查询订单详情表
4. **取消订单**: 检查订单是否存在 → 更新订单状态为已取消
5. **支付订单**: 检查订单是否存在 → 更新订单状态为待发货 → 更新支付状态为已支付
6. **确认收货**: 检查订单是否存在 → 更新订单状态为已完成

## 8. 注意事项

1. 所有接口都需要在请求头中携带JWT token进行身份验证
2. 订单操作（取消、支付、确认收货）需要验证订单是否属于当前用户
3. 创建订单时，订单详情列表不能为空
4. 支付订单只能操作待付款状态的订单
5. 确认收货只能操作待收货状态的订单
6. 取消订单只能操作待付款状态的订单
