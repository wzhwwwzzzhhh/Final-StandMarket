# 用户端秒杀活动接口文档

## 📋 文档概述

本文档详细描述了用户端秒杀活动相关的RESTful API接口，包括秒杀活动、秒杀券、特价商品和秒杀订单的完整接口规范。

**基础信息：**
- **服务地址**: `http://localhost:8080`
- **API版本**: v1
- **认证方式**: Bearer Token
- **数据格式**: JSON

---

## 🔐 认证说明

所有接口需要携带有效的JWT Token进行认证：

```http
Authorization: Bearer <your-token>
```

**Token获取方式：**
- 用户登录后获取
- 存储在localStorage中
- 自动通过axios拦截器添加

## 🔄 异步秒杀处理流程

### 完整业务流程

1. **用户浏览秒杀活动**
   - 调用：`GET /user/seckill/activity/list`
   - 获取所有可用的秒杀活动

2. **查看秒杀券列表**
   - 调用：`GET /user/seckill/coupon/list`
   - 获取活动下的秒杀券

3. **抢购秒杀券**
   - 调用：`POST /user/seckill/coupon/{couponId}/orders`
   - 立即返回订单号和状态信息

4. **轮询订单状态**
   - 调用：`GET /user/seckill/order/{orderNumber}`
   - 查询订单处理进度

5. **订单处理完成**
   - 状态变为1（待支付）
   - 跳转到订单支付页面

### 前端轮询逻辑

```javascript
async pollOrderStatus(orderNumber) {
  for (let attempt = 0; attempt < 20; attempt++) {
    try {
      const response = await seckillApi.getSeckillOrderByNumber(orderNumber)
      if (response.data.code === 0) {
        const order = response.data.data
        
        if (order.status === 1 || order.status === 2) {
          // 订单创建成功，跳转到订单页面
          this.$message.success('抢购成功！')
          this.$router.push('/order')
          return
        }
      }
      await new Promise(resolve => setTimeout(resolve, 1000))
    } catch (error) {
      await new Promise(resolve => setTimeout(resolve, 1000))
    }
  }
  
  this.$message.warning('订单处理超时，请稍后查看订单状态')
}
```

## 📊 错误码说明

| 错误码 | 说明 | 处理建议 |
|--------|------|----------|
| 0 | 成功 | 正常处理 |
| 1 | 参数错误 | 检查请求参数 |
| 2 | 认证失败 | 重新登录获取token |
| 3 | 权限不足 | 检查用户权限 |
| 4 | 库存不足 | 秒杀券已售罄 |
| 5 | 重复抢购 | 请勿重复提交 |
| 6 | 活动未开始 | 等待活动开始 |
| 7 | 活动已结束 | 活动已过期 |
| 8 | 限购已达上限 | 已达到购买限制 |

## 🔧 前端API配置

### axios配置
```javascript
import axios from 'axios'

const api = axios.create({
  baseURL: '/api',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器，自动添加token
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})
```

### 秒杀API模块
```javascript
export const seckillApi = {
  // 秒杀活动
  getSeckillActivityList: () => api.get('/user/seckill/activity/list'),
  getSeckillActivityDetail: (id) => api.get(`/user/seckill/activity/detail/${id}`),
  
  // 秒杀券
  getSeckillCouponList: () => api.get('/user/seckill/coupon/list'),
  getSeckillCouponDetail: (id) => api.get(`/user/seckill/coupon/detail/${id}`),
  seckillCoupon: (couponId) => api.post(`/user/seckill/coupon/${couponId}/orders`),
  
  // 秒杀订单
  getSeckillOrderList: () => api.get('/user/seckill/order/list'),
  getSeckillOrderByNumber: (orderNumber) => api.get(`/user/seckill/order/${orderNumber}`)
}
```

## 🚀 性能优化特性

### 1. 异步处理架构
- **高并发支持**: 前端立即响应，后端异步处理
- **数据一致性**: RabbitMQ保证订单最终一致性
- **系统稳定性**: 避免数据库操作阻塞请求

### 2. Redis缓存优化
- **库存控制**: Lua脚本原子操作防止超卖
- **限购验证**: 基于用户ID的购买次数限制
- **分布式锁**: Redisson防止重复抢购

### 3. 接口设计优化
- **RESTful风格**: 规范的接口路径设计
- **数据精简**: 异步场景下只返回必要信息
- **错误处理**: 完善的异常处理机制

---

## 📝 使用注意事项

1. **异步处理**: 抢购请求需要轮询查询最终结果
2. **库存限制**: 高并发场景下可能出现库存竞争
3. **时间控制**: 注意秒杀活动的开始和结束时间
4. **限购策略**: 每个用户有购买次数限制
5. **错误处理**: 需要处理网络异常和业务错误
