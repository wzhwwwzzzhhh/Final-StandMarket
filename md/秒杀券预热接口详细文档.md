## 🔥 预热单个秒杀券

- **请求方法**: POST
- **接口路径**: `/admin/seckill/coupon/preheat/{id}`
- **路径参数**:

- **请求体**: 无
- **Content-Type**: application/json


## 🔧 后端实现代码

### 控制器层实现
```java
@RestController
@RequestMapping("/admin/seckill/coupon")
public class SeckillCouponController {
    
    @Autowired
    private SeckillCouponService seckillCouponService;
    
    /**
     * 预热单个秒杀券
     */
    @PostMapping("/preheat/{id}")
    public Result<?> preheatCoupon(@PathVariable Long id) {
        try {
            seckillCouponService.preheatCoupon(id);
            return Result.success("预热成功");
        } catch (Exception e) {
            log.error("预热秒杀券失败，ID：{}", id, e);
            return Result.error("预热失败：" + e.getMessage());
        }
    }
    
    /**
     * 预热所有秒杀券
     */
    @PostMapping("/preheat")
    public Result<?> preheatAllCoupons() {
        try {
            seckillCouponService.preheatAllCoupons();
            return Result.success("预热所有秒杀券成功");
        } catch (Exception e) {
            log.error("预热所有秒杀券失败", e);
            return Result.error("预热失败：" + e.getMessage());
        }
    }
}
```

### 服务层实现
```java
@Service
public class SeckillCouponServiceImpl implements SeckillCouponService {
    
    @Autowired
    private SeckillCouponMapper seckillCouponMapper;
    
    @Autowired
    private StringRedisTemplate redisTemplate;
    
    @Override
    public void preheatCoupon(Long id) {
        // 1. 查询秒杀券信息
        SeckillCoupon coupon = seckillCouponMapper.selectById(id);
        if (coupon == null) {
            throw new RuntimeException("秒杀券不存在");
        }
        
        // 2. 验证秒杀券状态
        if (coupon.getStatus() != 1) {
            throw new RuntimeException("秒杀券状态无效");
        }
        
        // 3. 预热到Redis
        String stockKey = "seckill:coupon:stock:" + id;
        redisTemplate.opsForValue().set(stockKey, coupon.getStock().toString());
        
        // 4. 设置过期时间（秒杀结束时间）
        long expireSeconds = Duration.between(LocalDateTime.now(), coupon.getEndTime()).getSeconds();
        if (expireSeconds > 0) {
            redisTemplate.expire(stockKey, expireSeconds, TimeUnit.SECONDS);
        }
    }
    
    @Override
    public void preheatAllCoupons() {
        // 1. 查询所有有效的秒杀券
        List<SeckillCoupon> coupons = seckillCouponMapper.selectValidCoupons();
        
        // 2. 批量预热
        for (SeckillCoupon coupon : coupons) {
            try {
                preheatCoupon(coupon.getId());
            } catch (Exception e) {
                log.error("预热秒杀券失败，ID：{}", coupon.getId(), e);
            }
        }
    }
}
```

## 🔄 预热业务流程

### 预热单个秒杀券流程
1. **接收请求**：前端调用预热接口，传递秒杀券ID
2. **参数验证**：后端验证ID的有效性
3. **数据查询**：根据ID查询秒杀券信息
4. **状态验证**：检查秒杀券是否有效
5. **Redis预热**：将库存数据写入Redis
6. **设置过期**：根据秒杀结束时间设置Redis键过期
7. **返回结果**：返回成功或失败消息

### 预热所有秒杀券流程
1. **接收请求**：前端调用批量预热接口
2. **数据查询**：查询所有有效的秒杀券
3. **批量预热**：遍历所有秒杀券，逐个预热
4. **异常处理**：记录预热失败的秒杀券
5. **返回结果**：返回批量预热结果

## ⚠️ 注意事项

### 1. 安全性
- 预热操作需要管理员权限
- 需要验证秒杀券的存在性和有效性
- 记录操作日志，便于审计

### 2. 性能优化
- 批量预热时可以使用Redis管道操作
- 预热操作可以异步执行
- 避免在高峰期执行大量预热操作

### 3. 数据一致性
- 预热前确保数据库和Redis的数据一致性
- 预热失败时需要有回滚机制
- 监控预热操作的执行状态

### 4. 错误处理
- 提供清晰的错误消息
- 记录详细的错误日志
- 支持重试机制

## 🔍 接口设计原则

### 1. RESTful风格
- 使用POST方法执行操作
- 路径参数放在操作名称后面
- 统一的响应格式

### 2. 简洁性
- 请求体为空，只使用路径参数
- 响应只包含必要的信息
- 错误消息清晰明了

### 3. 可扩展性
- 支持单个和批量预热
- 易于添加新的预热功能
- 支持异步处理
